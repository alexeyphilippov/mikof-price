## 7. Критерии Pass/Fail (использовались при прогоне)

- **Pass UI** — элемент виден, статус/pill соответствует, данные корректны.
- **Pass API** — HTTP 2xx, ожидаемые поля в ответе.
- **Pass Email** — факт вызова `POST /send` в логах mailer (SMTP может вернуть `sent: false` локально).
- **Known Fail** — расхождение с ТЗ, зафиксировано как дефект/gap.
